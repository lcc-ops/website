一个 AI 工具实操账号,靠 Codex、Claude Code 教程和提示词案例涨到约 2.2 万粉。单条最高 1.5 万赞、播放超过 150 万。变现包括一份 19.9 元的《Codex 笔记》,自报卖出 169 份,以及远程协助服务。把数字算开后,它不是“爆款教程”故事,而是一条小额产品筛选高价服务客户的漏斗。

## 已披露的漏斗数学

| 指标 | 披露或推导值 |
|---|---:|
| 粉丝 | 22000 |
| 单条最高点赞 | 15000 |
| 单条最高播放 | 150 万+ |
| 笔记单价 | 19.9 元 |
| 笔记销量 | 169 份 |
| 笔记毛收入 | 3363.10 元 |
| 销量÷粉丝 | 0.77% |
| 每粉丝毛收入 | 0.153 元 |

0.77% 不是严格转化率。部分买家可能没有关注账号,很多粉丝也可能根本没看到销售内容,粉丝数还可能在销售期后继续增长。但它可以做量级检查:一个 2.2 万粉账号,没有靠 19.9 元文件变成大型信息生意。披露的文件毛收入是 3363.10 元,还没扣费用和支持成本。

这不代表文件没有价值,而是它的工作不同。笔记把宽泛的教程流量变成 169 个愿意为 Codex 问题付钱的人。下一层若是远程安装、调试、工作流设计或团队培训,这些买家比匿名播放更值钱。

## 教程承接的是问题,不只是观众

Codex 和 Claude Code 内容自带商业信号。看一次提示词演示的人可能只是好奇。下载笔记、复制命令、追问为什么环境跑不起来的人,已经有实施问题。

账号自报内容以具体教程和提示词案例为主,不是泛 AI 新闻。这个差别很重要。泛新闻能拿大流量,但购买意图弱。某个具体任务的教程流量更小,却暴露了一个付费节点:有人愿意花钱省掉配置时间和失败成本。

这条内容漏斗有三层:

1. **免费证明。** 短教程证明运营者能解决一个窄问题。
2. **低价压缩包。** 19.9 元笔记把命令、提示词和流程包装成自助参考。
3. **付费介入。** 远程协助处理环境问题、定制工作流和文档无法覆盖的情况。

第一层拿注意力,第二层记录购买意图,第三层可能承担主要利润。如果没有第三层,披露的笔记需要大很多销量才会成为有意义的收入。

## 19.9 元能承担什么,不能承担什么

19.9 元的客服经济很苛刻。若每位买家只问一个 10 分钟问题,169 人会消耗 28.2 小时。还没扣平台费、制作、退款和税,每客服小时对应毛收入约 119 元。若平均支持时间达到 30 分钟,这个数字会降到 40 元/小时以下。

所以文件必须有硬边界:固定内容、明确更新政策、不包含个性化调试。远程帮助要作为另一项付费服务单独定价。

用 169 位买家做简单测算:

| 169 位买家的服务转化 | 服务订单 | 单价 | 服务收入 |
|---:|---:|---:|---:|
| 3% | 5 | 299 元 | 1495 元 |
| 5% | 8 | 499 元 | 3992 元 |
| 10% | 17 | 999 元 | 16983 元 |

这些不是案例实绩,只是情景。它说明缺失的远程服务数字比“169 份”更重要。若 5% 买家以 499 元成交,服务收入已经超过笔记毛收入。若 10% 以 999 元成交,服务层大约是笔记毛收入的 5 倍。

## 成本结构和单位经济

数字文件的交付边际成本接近零,但经营成本不是零:

- **内容研究。** 命令和界面变化快,示例要反复重测。
- **包装。** 截图、结构、校对和版本标记都要时间。
- **支付与平台费。** 小额商品会被固定费用或比例费用吃掉明显一块。
- **退款与盗版。** 可复制文档很容易离开付费渠道传播。
- **客户支持。** 低价买家也可能提出高触感问题。
- **更新责任。** 绑定快速迭代工具的产品很快会过时。

假设平台费和退款合计吃掉毛收入 10%,人工前净收入约 3026.79 元。若笔记制作和更新花 20 小时,前 169 份销量对应每制作小时 151 元,还没有扣客户支持。作为可复用资产可以接受,但单独看不是大生意。

内容账号本身也有生产成本。研究、录屏、剪辑和回复可能比写笔记花更多时间。爆款播放会把这块劳动藏起来,因为平台页面没有成本行。

## 真正有用的指标不是粉丝数

2.2 万粉和 150 万峰值播放证明账号有分发,但不能证明漏斗稳定。更有用的测量链是:

```text
教程播放
→ 主页访问
→ 商品页访问
→ 完成笔记购买
→ 买家发起求助
→ 远程服务成交
→ 复购或转介绍
```

每一步回答不同问题。播放高、主页访问低,说明内容只提供娱乐,没有产生意图。商品页访问高、购买低,问题可能在包装或信任。购买后没有服务咨询,若文件本身有利润也没问题,但它不支持服务模型。服务成交后没有复购,可能交付太窄或质量不稳。

教程账号还会遇到受众错配。最容易拿赞的可能是入门提示词,而真正愿意付远程协助的人,面对的是复杂本地环境或公司约束。运营者必须决定内容是追求最大触达,还是筛选正确问题。

## 案例没有覆盖什么

- **没有总收入。** 远程协助被提到,但没有单价和订单数。
- **没有时间窗口。** 不知道涨到 2.2 万粉和卖出 169 份用了多久。
- **没有完整漏斗。** 商品页访问、结账完成、退款和客服量都缺失。
- **没有内容总量。** “每 1–2 条出一个爆款”没有总样本。
- **没有成本。** 录制、剪辑、更新、支付费用和客服都没算。
- **没有留存。** 一次性笔记和一次远程服务不会自动形成连续收入。
- **没有保鲜政策。** Codex 和 Claude Code 变化很快,旧说明会增加退款与支持。

1.5 万赞尤其容易被高估。大批点赞可能来自从不购买的泛受众。169 次付费交易虽然小得多,商业信号反而更强。

## 带走就一句

已披露的笔记是一款毛收入 3363.10 元的小额产品,不是大型知识付费生意。它的战略价值在于把教程观众变成 169 位有明确 Codex 问题的付费买家。若远程协助单独定价,并且账号吸引的是实施需求而不是泛 AI 好奇,服务层可能比文件值钱。

下一步测试不是再追一个爆款,而是把漏斗量出来:固定一种教程格式、一页笔记销售页、一项清楚标价的远程服务,记录从播放到服务回款的每一步。这样才能判断账号拥有可复用获客渠道,还是只有一次注意力脉冲。
