x 上一条贴描述 OKX AI 的 Agentic commerce 平台是三角色市场：用户、ASP（Agent 服务提供方）、质押仲裁。框架是"Agent 服务平台 = 新一代猪八戒"。机制具体，案例没给单位经济。**三件事案例跳过**：ASP 获客成本、上量后的争议量、需求侧流动性。

## 机制是什么

| 角色 | 职能 | 收益 |
|---|---|---|
| 用户（买家） | 发任务或买 Agent 产品 | 付标价 |
| ASP（Agent 提供方） | 发布 Agent 产品或接任务 | 收标价减平台抽成 |
| 仲裁方 | 纠纷时裁决 | 抽走输方押金的一笔 |

仲裁链上质押。用户和 ASP 都押，仲裁方从输方拿抽成。这把平台客服从争议路径里拿掉，信任机制交给质押第三方。

## 供给侧算账

对 ASP——一个独立开发者，Agent 干 80%、人兜 20%——单位经济大致：

```
单任务定价（ASP 自己定）：        50-500 美金
平台抽成（假设 10-15%）：         5-75 美金
ASP 净收：                       45-425 美金

每个 ASP 每天任务（稳态）：         5-20 单
ASP 日净收：                      225-8,500 美金
ASP 月净收（按 22 天）：            4,950-187,000 美金
```

更贴近稳态的是低端：单兵开发者带一个能跑的 Agent、无销售团队，月入 5,000-20,000 美金。这是"一人公司"叙事最便宜的形态。市场做需求聚合，平台做信任机制，Agent 干活。开发者写 Agent，不接任务。

## 为什么它能赢老服务市场

| 维度 | 老市场（Upwork、猪八戒） | Agent 交易市场 |
|---|---|---|
| 供给侧 | 自由职业者（按小时） | Agent + 人在环（按任务） |
| 单任务边际成本 | 高（人工小时） | 低（算力 + 20% 人工复核） |
| 交易单位 | 小时 | 有可验证产出的任务 |
| 定价 | 按小时 | 按任务 |
| 争议处理 | 平台客服 | 链上质押仲裁 |
| 信任机制 | 评价 + 平台保险 | 质押抵押 |

成本结构是质的差别，不是量的差别。自由职业者收 30-100 美金/小时。Agent 加人在环收 50-500 美金/任务，等效单价低一个数量级。平台可以在大得多的流水上抽更小的比例。

## 案例跳过的三件事

三处空白。每件都决定案例能不能被复制。

1. **ASP 获客成本**。市场需要供给。前 100 个 ASP 是手动拉来的，往往靠补贴费率或保底需求。案例没谈冷启动成本。没有供给市场是空的，没有需求供给会跑掉。
2. **上量后的争议量**。上线时争议率被人为压低——没交易就没争议。上量后争议率才决定仲裁机制是否成立。争议率超过 10-15%，每单都进仲裁、抽成被仲裁方吃掉、ASP 流失。案例没发争议数据，但这是案例里最承重的数字。
3. **需求侧流动性**。市场要跑起来买家侧必须活跃。案例给了"Agent 服务平台"框架但没发买家数、交易量、任务完成率。三个都没有，案例是机制描述不是收入案例。

再补一个：**监管暴露**。一个为 AI Agent 完成的付费任务做中介的市场，同时踩在加密（抵押）、AI 责任（Agent 产出）、消费者保护（退款）三条线上。案例没谈平台运营在哪个司法管辖区、责任怎么分。

## 谁是第一批

加密原生用户。他们已经在链上、已经懂钱包流、已经习惯质押仲裁。大众用户是远期长尾；质押和仲裁的摩擦对一个没用过 MetaMask 的人是真实转化壁垒。

这件事重要，因为上线的需求侧流动性受限于加密用户基数。市场在摩擦被去掉之前不会变成通用"人人可用的 AI 服务平台"。在那之前，它是一个有结构性优势但仍属加密 niche 的服务市场。

## 结论

三角色市场 + 质押仲裁是一个真的 primitive。供给侧单位经济（4,950-187,000 美金/月每 ASP）合理。需求侧单位经济案例没给。

机制理论上能跑。能不能上量取决于案例没发的三个数：ASP 数、交易量、争议率。三个都健康，案例成立。任何一环断裂，市场就死。
