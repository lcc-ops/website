faq:
  - q: "他到底卖的是什么?"
    a: "给牙科、暖通、房产、保险客户做的「简单AI自动化」。 一个典型订单就是一条工作流: 新线索进来 → 系统判断意向 → 90秒内自动写个性化回复邮件。 整套搭在 Zapier + GPT + 一个 CRM hookup 上。 安装费3000-7000美金, 月费500-1500美金覆盖监控和 prompt 调整。"
  - q: "关键数字是什么?"
    a: "年收入75000美金; 18个客户; 平均单价4200美金; 月费收入占60%(8个客户在付, 其中3个超过8个月没断); 2个月费客户为他带来11000美金的转介绍新单, 营销费0。"
  - q: "为什么月费才是命脉?"
    a: "三个原因。 第一, 不同订单的搭建时长差很大——12小时和40小时的订单收一样的安装费, 后者等于打零工。 第二, 搭完之后客户还要 prompt 调整、CRM 接口坏掉排查、静默故障监控, 月费把这些都装进去。 第三, 凌晨两点 API 静默断线的时候, 操作者是那个在客户醒来前就修好的人, 这种「醒来前修复」的承诺构建了切换成本, 一次性收费模式永远建不起来。"
  - q: "为什么专挑无聊行业?"
    a: "因为那些老板没有内部技术团队, 又被手动跟进和排班淹死。 他们不会发 RFP, 不会比三家, 不会问用哪个模型哪个编排器。 决策标准是「这事能不能消失」, 不是「技术方案谁最强」。 这种决策结构让自由职业者能拿一页纸的结果陈述把单签掉, 整个采购流程直接跳过。"
  - q: "AI 在这单里到底怎么压毛利?"
    a: "用 AI 之前, 一个 Zapier 加 GPT 的部署要一周。 用 AI 之后, 一个周末。 客户感知到的价值没变——他们省下来的人工时长、首响时间、漏单损失都没变——所以收费不变, 交付成本压缩了一半。 差值就是毛利。"
  - q: "这个案例没说的事?"
    a: "六条缺口。 第一, 退款和纠纷率——2500-7000美金安装费不算小数, 这一类业务月费纠纷很常见, 案例作者没公布。 第二, API 成本占毛利的份额——Zapier 订阅 + GPT API 算在操作者头上, 一个自定义 prompt 跑 GPT-4o 级推理吃毛利, 案例没拆。 第三, 续费曲线——只有3个客户「超过8个月」这一个数据点, 12个月之后是什么样完全不知道。 第四, 容量上限——每周<15 工时, 每月新签到第几个就过载? 第五, 行业防御性——牙科诊所和暖通店转向 SaaS 化竞品是2027年的现实威胁。 第六, 个人工时风险——案例默认是单人操作, 没有备份, 一个月生病就是零交付。"
