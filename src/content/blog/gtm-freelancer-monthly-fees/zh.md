一个 GTM 接单的自由职业者今年早些时候做了一单: 客户随口问「能不能搞个 AI 帮我跟进线索」, 他报价2500美金, 周末两天用 Zapier + GPT 搭好, 客户的销售团队首次回复时间从14小时压缩到3分钟以内。 客户跟朋友提了一嘴, 朋友也打电话来了。 这条转介绍链子就是他一年做下来75000美金、18个客户总盘子的来源。

这个案例真正有意思的不是搭建本身, 也不是工具, 是他在那之后做的四个不同选择。

## 数字

```
年收入:                              $75,000
年内活跃客户数:                       18
平均单客户总价:                       $4,200
月费收入占年收入比例:                 60%   (8 客户在付月费)
最长续费客户:                         3 个 (超过8个月)
2个月费客户转介绍:                    $11,000 (营销费0)
安装费区间:                           $3,000 - $7,000
月费区间:                             $500 - $1,500
首单周末搭建收费:                     $2,500
```

这一组数字里最关键的是「60%是月费」。 18 个客户里, 8 个在持续付月费, 其中 3 个超过8个月没断——这个结构让一个自由职业看起来更像是一个微 SaaS, 不像接单。

## 案例里的四个反常识模式

第一, 卖结果, 不卖工具。 没有任何一个客户问过他用什么模型、跑的是 n8n 还是 Make 还是 Zapier。 客户只问: 线索多久能收到回复? 团队一周能省几小时? 跟进率能涨多少? 把话术从「我给你搭一个 GPT 驱动的多步自动化工作流」换成「你的线索会在90秒内收到个性化回复, 7×24小时」, 成交率直接起飞。

第二, 专挑无聊行业。 他最赚钱的客户不是科技公司, 是牙科诊所、暖通维修、房产团队、保险经纪人。 这些行业的内部技术人才数是零, 全靠手动跟进和排班淹没。 他们不会要技术方案书, 只想让问题消失。

第三, 越简单越赚。 这个操作者做过30多个自动化, 规律永远是越简单活得越久。 一个多 AI 对话 + 知识库 + 推理面板的复杂系统, 开发6周, demo 拿了 LinkedIn 赞, 三分之一回答是错的, 3个月后项目死了。 一个「每天早上醒来找新线索 → 给每个人写个性化邮件 → 扔进表格等发送」的脚本, 5天搞定, 跑了8个月, 客户一次修改都没提过。

第四, 月费才是命脉。 他前期全是一次性收费, 2500美金交付完事。 问题是: 搭12小时的项目和搭40小时的项目收一样的钱, 后者等于打零工。 后来改成「3000-7000 安装 + 500-1500 月费」之后, 收入与工作量对齐了。 月费覆盖 prompt 调整、API 接口坏掉排查、凌晨静默故障修复。

## 单位经济拆细

```
安装搭建工时:                          12 - 40 小时
Zapier 订阅 (算操作者):                  ~$20-50 / 月
GPT 级 API 费用 (每次搭建):              $5 - $40
安装费毛利:                              ~85 - 90%
月费单客户单月工时:                      1-3 小时
月费毛利:                                ~70 - 80%
首年单客户价值:                          $4,200
第二年起单客户价值 (无安装费):           $6K - $18K (取决于月费拉多长)
```

第一年装安装费里有85-90%的毛利, 第二年起单客户只剩月费, 操作者每月只花1-3小时, 单小时收入折到$200+/小时——这才是这个生意结构的真正算账。

## 这个案例没告诉你的事

六条缺口。 第一, 退款和纠纷率——$2,500-$7,000 的安装费不算小数, 这一类业务月费纠纷常见, 案例作者没公布。 第二, API 成本的占比——Zapier 订阅 + GPT API 都算操作者支出, 一个自定义 prompt 跑 GPT-4o 级推理吃毛利, 案例没拆出来。 第三, 续费曲线——3 个客户超 8 个月只是一个数据点, 12-24 个月的曲线是完全未知的。 第四, 容量上限——每周<15 工时, 每月新签到第几个会撞顶? 第五, 行业防御性——牙科诊所 / 暖通店被 SaaS 化竞品接管是 2027 年的现实威胁。 第六, 个人工时风险——案例默认单人操作, 没有备份, 一个月生病就是零交付。

## 带走就一句

这个案例的数据点不是周末搭建的2500美金, 是「第二年8个客户里的3个撑了8个月, 月费收入占总盘60%」——这个结构让一个接单变成一个接单+订阅的混合体, 而不需要融一分钱。
