一支做消费 AI 修图和视频的小团队,2025 年营收翻了 14 倍,年化冲到 5500 万美金,2026 年初推到 7000 万美金。窗口期内产品没做大版本模型升级,增长的引擎基本是一项运营决策:多数付费漏斗从 App Store 搬到了自建网页。下面把案例摆在桌面上的数字拆开,以及四点案例没说的事。

## 数字

| 数量 | 值 |
|---|---|
| 2025 营收增长 | 14 倍 |
| 年化营收(2025) | 5500 万美金 |
| 年化营收(2026 初) | 7000 万美金 |
| App Store 类别排名 | 照片视频类前 5 |
| 融资 | 无,自负盈亏 |
| 风格模板数 | ~1000 |
| 累计成片 | ~10 亿 |

数字为公司自报、未审计。用作方向性论据,不是财报。

## 支付通道的数学

案例把抽成算得很具体:

```
7000 万美金走 App Store   →  到手约 4900 万美金(扣 30%)
7000 万美金走自建网页      →  到手约 6650 万美金(综合扣约 5%)
                              差额 ≈ 1750 万美金/年
```

这是故事的核心数字。案例说得很直白:同样的用户、同样的产品,只是换了收款的门。1700 万美金的差额,比大多数 AI 消费团队一季度花在模型 API 上的钱都多。

紧挨着抽成的,还有一条隐性的运营成本:税务与退款。在 App Store,苹果代收 VAT、代缴销售税、处理退款与拒付;搬到自建网页,这些活转给运营方自己扛。7000 万美金 GMV 之下,这是一条真实的人力线,案例没点名,但言下之意是已经招了人来扛。

## 为什么 App Store 是大多数 operator 忘记交的税

大多数消费 AI 团队优先上 iOS,因为 App Store 是获客最便宜、但单位经济学最差的渠道。免费安装零摩擦,但每一次付费转化都要给苹果 30%,运营方看不到是谁付的、什么时候付的、这个同期群后面怎么走。平台掌握客户关系。

案例描述的是另一种打法:iOS 负责留存与日活,网页负责获客与付费转化。广告把流量引到自建定价页,用户在网页上付完款,拿到产品访问权限,之后才被邀请去下 App 做更重的体验。App 是日常驾驶舱,网页是结账台。

这条切分有两个结构性好处:

1. **运营方掌握付费客户。** 邮箱、账单、流失原因、退款原因、扩张信号——运营方全可见。苹果不允许这些数据按同期群粒度导出,而且能导出的也有延迟。
2. **广告 CAC 可量化。** 某个关键词带来 50 个付费网页客户、CPA 30 美金,可以加预算。某个关键词带来 200 次 App 安装、CPA 6 美金但零订阅,应该立刻停。案例说正是这套数据,让公司一年内把苹果搜索广告的预算放大了 13 倍。

## 「不做模型」的姿态

案例里第二个不太显眼的取舍,是团队不训基础模型。产品是把大约 1000 个预设「风格」搭在第三方图像和视频模型 API 之上,商业 API 加少量开源都接。每个预设是一条一键工作流——用户描述需求,系统选模型、应用风格、输出成片。

这是一笔主动的交换。运营方不付训练基础模型的资本开支、不背推理成本的长尾、不用每两个月追一次前缘。代价是没有模型护城河。他们真正在堆的护城河是 1000 条工作流的目录、10 亿次成片的数据,以及网页结账台的转化闭环。

案例说得很直白:用户看不到模型参数,只看到结果。这家公司对外讲的心智模型是「三分钟从提示词到成片」,不是「哪家的扩散模型更适合这个场景」。

## 案例没说的事

案例把团队做了什么讲得具体,能不能照搬到另一支团队,关键看四点:

1. **模型 API 之后的毛利。** 接第三方模型比自训便宜,但 API 账单按调用次数跑。10 亿次成片之下,这条成本线不小,而且很可能是最大可变成本。案例没给单位经济学表。
2. **苹果对站外跳转的政策风险。** 苹果多次诉讼过把用户引向站外结账的设计。案例没说团队怎么处理政策风险,一旦政策改,漏斗一夜要重做。
3. **退款与拒付的运营开销。** 自建网页意味着团队自扛争议处理。7000 万美金 GMV 按行业平均 0.5–1% 的拒付率,是 35 万–70 万美金/年的运营工作量与潜在损失。
4. **销售税与 VAT 合规。** 自建网页要在每个用户付款的司法管辖区代收代缴。全球体量下,这是真实的合规线和真实的工程线。案例两条都没点名。

支付通道这件事是真金白银、且金额大。能不能搬到更小的 App 上,主要看运营方愿不愿意把苹果商店现在吸收掉的合规与政策风险接过来自己扛。
