一个国人 operator 在 GitHub 上整理了 5 个开源 AI 副业剧本。合集覆盖 AI 工具、SaaS、自媒体、API 聚合、海外项目、AI 内容制作、AI 图像服务、AI 短剧制作。案例不是单个剧本——是对 2026 年 AI 副业剧本「反复出现的形状」的元解读。每个剧本出货:一张工作流图、一个工具清单、一条陡得不合理的「营收曲线」。5 个里 4 个缺单位经济部分。元教训:剧本出货上行,跳过缺口,大多数读者在缺口里亏钱。下面是反复出现的形状、单位经济缺口、以及案例没展开的四点。

## 5 个剧本反复出现的形状

```
剧本章节                       出货了吗  备注
──────────────────────────────────────────────────
工作流图                       出货      输入 → AI → 输出 → 分发
工具清单                       出货      模型、自动化、支付
营收曲线                       出货      线性增长,没见顶
成功故事(第三方)               出货      不是作者自己的营收
CAC 章节                       没       ──
转化率章节                     没       ──
退款/拒付章节                   没       ──
衰减/长尾章节                   没       ──
中位结果章节                    没       ──
失败模式章节                    没       ──
```

形状在 5 个剧本里一致。结构论点不是剧本写得不对——是剧本写得不全。剧本出货上行,跳过缺口。缺口就是单位经济风险待的地方。

## 单位经济缺口

```
剧本声称:                        现实(operator 一侧):
- 线性增长                       - 70% operator 90 天内 0–1K 美金
- AI 干所有活                    - AI 干 60–80%,人干 prompt 设计、
                                   质量控制、分发
- 工具都列了                      - 工具都列了;价格和续费没
- 引用了成功故事                  - 中位结果没引
- 没提 CAC                       - CAC 是主要成本
- 没提转化率                     - 免费到付费转化率 1–5%
- 没提退款率                     - 退款率 5–15%
- 没提衰减                       - 算法变化 3–6 月营收衰减
```

缺口是结构性的。剧本假设无摩擦的分发渠道;现实是分发是瓶颈。剧本假设工具栈免费或近免费;现实是工具栈每月复利成本 200–2000 美金(规模化下)。

## 为什么剧本对头部 5–10% 有效、对中位数不奏效

剧本对有以下条件的 operator 有效:
- 一个分发渠道(现有受众、付费流量或内容引擎)
- 愿意迭代 prompt 设计与质量控制
- 财务跑道撑得住 3–6 月负现金流

对这些 operator,剧本是一个起始模板,他们调整并执行。对中位数读者——以上都没有——剧本是一个菜谱,产出几周活动,然后放弃。

结构论点是:剧本是写给 operator 的,不是写给读者的。按剧本照搬的读者在缺口里亏钱。把剧本当起始模板、补齐单位经济缺口的读者,有机会吃上行。

## 案例没说的事

四点。

1. **作者营收。** 剧本没给作者自己的营收,只整理第三方成功故事。作者自己的结果是核心证据;没有它,剧本是理论的。
2. **中位结果。** 没报中位结果,只报分布顶部。1–5% 的「成功率」合理但没说。
3. **失败模式。** 失败的剧本没被收录,只收录跑通的。生存偏差放大了暗示的成功率。
4. **剧本对头部 5–10% operator 有效、对中位数读者无效这个结构性论点,案例没做。** 案例把合集框定为正面资源;缺口是那个没说出的部分。

## 结论

案例不是「GitHub 上 5 个 AI 副业剧本」。案例是:2026 年 AI 副业剧本生态出货上行,跳过单位经济缺口。剧本对头部 5–10% 的 operator 有效(他们本来就有分发);对中位数读者不奏效。缺口是大多数读者亏钱的地方。

看这些剧本的多数读者,带走就一句:把剧本当起始模板看,不是菜谱。出货工作流之前,先补齐单位经济缺口。上行对有分发的 operator 是真的;对其他人是理论的。
