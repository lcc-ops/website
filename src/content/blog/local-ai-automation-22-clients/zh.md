faq:
  - q: "产品具体是什么?"
    a: "一条挂在客户网站表单后的 n8n 工作流, Claude 读取用户提交的内容（预算/时间/地点）, 判断高意向还是低意向, 高意向直接推日历预约链接, 低意向发送一封有用的跟进邮件。 每个客户搭建4小时, 月费覆盖监控、Prompt 调整、以及 API 静默故障排查。"
  - q: "定价结构怎么算?"
    a: "每个客户一次性3500美金安装费 + 每月300美金维护费。22个客户的维护费就是6600美金/月的固定收入。 加上每月新签3到4个客户（每个3500美金一次性）, 又多10500到14000美金。 两者合计每月稳定在21000美金以上, 每周实际工时不到15小时。"
  - q: "为什么是自由职业, 不是 SaaS?"
    a: "三个结构原因。 第一, 客户是完全没有时间学 n8n 的本地小老板, 他们不会自助, 永远不会。 第二, 每个客户的工作流都定制 (CRM、日历、线索源不同), 本身就阻止了自助产品。 第三, 因为 Claude 能从自然语言写 n8n 的 JSON 配置, 单个客户的边际时间压到4小时, 在自由职业的毛利率下赚钱, 在 SaaS 的获客成本下不赚钱。"
  - q: "客户从哪来?"
    a: "本地商会活动、Yelp 搜索「50条以上评论 + 3.5-4.0星」的商家（这是「有客流但转化烂」的标准信号）、Facebook 本地商业群组。 没有作品集网站, 不做 SEO, 不写内容营销。 前5个客户全部来自一个商会网络的人脉转介绍。 Upwork 上确实有同行的自由职业者时薪50美金, 但他们不会去找街边的牙医和水管工。"
  - q: "AI 在这件事里到底做什么?"
    a: "Claude 做的是实现工作, 不是面对客户的工作。 操作者用自然语言描述需求, Claude 输出 n8n 的 JSON 节点配置——这个动作在没有 AI 的时候至少20小时手工写 JSON, 有 AI 之后4小时。 客户面对的那层 AI (线索打分) 是一个薄包装, 真正的杠杆在搭建时间的压缩, 不是模型的智能。"
  - q: "这个案例不告诉你的事?"
    a: "五条缺口。 第一, 流失率——案例作者没公布月度客户取消率, 22个客户存量不告诉你每年走多少人。 第二, 付款节奏——3500是一次性给还是分期?影响现金流和坏账风险。 第三, 单项目毛利——Claude API 成本、n8n 托管、未计入的1-2小时月度支持工时都没拆出来。 第四, 地域——这是美国 SMB 市场, 愿意付费额度、SaaS 生态都和国内不一样, 直接平移会撞墙。 第五, 防御性——22个客户的存量, 明年如有同行用同样的本地渠道低价抢, 没有任何护城河。"
