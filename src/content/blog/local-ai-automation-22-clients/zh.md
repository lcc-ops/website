一个在美国的自由职业者, 此前做 WordPress 建站, 时薪40美金封顶。 2025年底他转向了一份新报价: 用 n8n 加 Claude 搭 AI 线索筛选工作流, 卖给那些没有内部技术人员的本地服务类商家。 到2026年6月, 他手上22个客户, 每月进账超过21000美金, 自己每周实际工时不到15小时。

这件事有意思的杠杆不是 AI 模型, 而是一套模板在多个行业之间复用。 工作流的结构是「网站表单提交 → Claude 评估意图 → 高意向推日历预约链接, 低意向发一封有用的跟进邮件」。 这条流程搬到牙科诊所、暖通维修、房产经纪、律师、清洁公司, 都是一个新付费客户。 上线的功能数量是一个。

## 数字

```
单客户一次性搭建费:                 $3,500   (安装 n8n + Claude 工作流)
单客户月度维护费:                   $300     (监控、Prompt 调整、API 故障修复)
固定月度收入 (维护费合计):         $6,600   (22 客户 × $300)
每月新增签约客户:                   3 到 4   (每个 $3,500 = $10.5K-$14K)
每月总进账:                        $21K+    (维护费 + 新单)
单客户搭建耗时:                    4 小时    (Claude 写 n8n JSON 配置)
每周实际工时:                       < 15 小时
搭建折合时薪:                       ~$875/hr  ($3,500 / 4h)
```

「每周实际工时 <15」这一行是案例作者自己说的, 不是从收入反推; 在算账的时候要和月度新增客户的签字时间放一起看。

## 为什么是这个组合: n8n + Claude

工具组合的选择不是「任意工作流工具 + 任意大模型都能换」。 n8n 提供可视化节点图 + JSON 导出, Claude 直接写 JSON, 操作者手工调试。 Make.com 和 Zapier 也能跑同样结构, Zapier 对非技术人员体验更顺, n8n 在规模化时更便宜且支持自托管。 模型层 (Claude) 可替换, 工作流工具可替换, 但结构不可换——一套模板, 多个行业。

模板能打中, 是因为三个原因。 第一, 目标买家是听说过 ChatGPT 但不会把它接到工作流的小老板, 永远不可能自助, 也不会读30页的技术方案书。 第二, 线索筛选这个结果很具体, 演示成本低——「你网站表单的线索, 90秒内收到个性化回复, 7×24小时」。 第三, 操作者获客渠道是「本地」——商会活动、Yelp 信号、Facebook 本地商业群组——Upwork 上时薪50美金的同行不会去扫这些渠道。

## 单位经济拆细

```
搭建成本 (操作者4小时, 不显式收费):     收入 $3,500
搭建期间 Claude API 成本:                ~$5-15 每客户
搭建期间 n8n 托管成本:                   ~$0 (自托管或免费层)
月度维护费收入:                          $300 / 月
单客户月度平均支持工时:                  1-2 小时
单客户月度毛利:                          ~$240-280 / 月
新客户首月现金流:                        $3,500 一次性 + 首个 $300
老客户第2个月开始:                       $300 / 月纯维护
22 客户 × $300 月度锁定:                 $6,600 / 月
```

一个反直觉的数据点: 操作者不另外收超出维护费的工时。 客户的临时问题消化在每月300美金里。 如果月度支持工时超过1-2小时/客户, 毛利就会被吃掉。 案例作者自己承认这是约束——「越简单的工作流, 工作才越好做」是他反复强调的规则。

## 这个案例不告诉你的事

五条缺口。 第一, 流失率——操作者没公布客户保留率, 22个客户的存量数字不告诉你每年走多少人。 第二, 付款节奏——$3,500 是首付? 50/50? 还是首月维护费抵? 影响现金流和坏账风险, 案例没提。 第三, 单项目毛利——Claude API 成本、n8n 托管、未计入的1-2小时月度支持工时没拆出来; 表面时薪875美金, 净时薪另算。 第四, 地域——这是美国 SMB 市场, 客户付费意愿、CRM 生态都和中国不一样, 直接照搬会撞 WeChat 替代 CRM、付费意愿低一半这两堵墙。 第五, 防御性——22客户存量, 没有护城河——明年如有同行用同样渠道 + 低价抢, 没有 switch cost。

## 带走就一句

结构性的杠杆是「同一个工作流卖22次, 每次只换行业话术」, 比同一个人做软件产品更赚钱。 复利在客户数, 不在功能数。
