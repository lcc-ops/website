# 订单履约 SOP — 端到端

一气呵成：接单 → 采购 → 打包 → 发货 → 签收 → 售后。括号里的时效是单店 / 小团队的真实值，不是日单 5000 的 3PL 节奏。

## 什么时候用

- 新店前 90 天。可选步骤先跳过，等单量逼你做再补。
- 接入新 3PL 或换新承运商。
- 延迟率 > 8% 或退款率 > 6%（剔除无理由退货）。
- 每季度：根据实际数据刷新阈值。

## 阶段 1 — 接单（T+0）

下单后 15 分钟内：

- [ ] 订单自动进 OMS（Shopify、WooCommerce、自建）。
- [ ] 地址按承运商规则校验（USPS、Royal Mail、菜鸟等）。
  拒收：UPS 走 PO box、缺邮编、不到的区域。
- [ ] 风控：客单价 3 倍以上、新邮箱下单 → 人工审。
- [ ] 库存锁定（扣可用数，不扣在途）。
- [ ] 买家确认邮件发出，承诺送达窗口，不承诺具体日期。

如果命中风控 → 挂起，不进入采购。

## 阶段 2 — 采购（T+0 到 T+1）

按 SKU 备货周期选路径：

| 备货周期 | 动作 |
|---|---|
| 现货 | 直接从仓库出，不动供应商 |
| 3–7 天 | 立刻下供应商单（AliExpress、货代） |
| > 7 天 | 进等待队列；买家不接受延后则退款 |

- [ ] 供应商单号绑到买家订单号。一行一个 SKU。
- [ ] 供应商确认后 24 小时内，把可追踪链接回写 OMS。
- [ ] 备货比原承诺晚 > 2 天，主动通知买家。

这一阶段不做的事：不管供应商合同、不管补货阈值，那是库存规划。

## 阶段 3 — 打包（T+1 到 T+3）

- [ ] 按可用库存打印拣货单。
- [ ] 质检：每件查破损、缺配件、错规格。单批拒收率 > 2% → 升级到采购部，不升级客服。
- [ ] 按发货规则打包（重量、易碎、危包）。不要盲信 SKU 重量字段 — 改了尺寸要重新称重。
- [ ] 按 A/B 实验计划放插卡。记录每个订单放了什么插卡。
- [ ] 按 SLA 选承运商和服务等级，不是按最低价。

日单 > 200 件按 SKU 分线。否则一个队列就行。

## 阶段 4 — 发货（T+3）

- [ ] 承运商取件确认或自送签收扫描。
- [ ] 标签扫描 1 小时内把追踪号推到 OMS 和买家邮件。
- [ ] 追踪号走主流聚合器（17track、AfterShip）。不要发承运商原始链接，发出去像垃圾邮件。
- [ ] "已发货" 通知要写：预计到达区间、客服联系方式、退货政策链接。

不要承诺到达日期，承诺到达窗口。天气、承运商异常、节假日会破日期。

## 阶段 5 — 在途（T+3 到 T+ETA）

只做两个检查，剩下的交给自动化。

- [ ] 每日异常扫描：> 5 天无扫描的订单 → 人工查询。
- [ ] T+7（或 ETA 一半，取较早）还没签收，主动发"在路上"邮件。能减少 ~30% 的"我的包裹呢"工单。

到此为止。别去承运商后台"盯包裹"，只会徒增噪音。

## 阶段 6 — 签收与售后（T+ETA）

- [ ] 承运商扫描 48 小时内触发签收确认。没触发按"可能丢件"处理 → ETA 后第 7 天起算索赔。
- [ ] 签收后 7 天发 1 个问题的评价请求。一个问题。开放评论框可选。
- [ ] 退货：政策简短，流程清楚。预打印退货标签 = 更少"怎么退"工单。
- [ ] 退货收到后 48 小时内退款，不是收到退货请求后 48 小时。

## 时效汇总

```
接单      15 分钟    自动化为主
采购      ≤ 24 小时  当班下供应商单
打包      1–2 天    取决于仓库位置
发货      当天      承运商截单前
在途      5–14 天   承运商控制
售后      ≤ 14 天   评价请求触发
```

任何阶段持续超时，靠流程修，别靠人海。

## 本 SOP 不覆盖

- 退货处理细节（退款时效有，退货质检分级没有）。
- 3PL 合同谈判。自发货搞不定了再谈。
- 跨境退货清关。
- 订阅 / 续费订单。规则不同。
- 投诉处理话术。这一层是履约层。

## 来源

从单店 / 小团队的真实运营复盘整理，2024–2026。
阈值数字来自实际订单复盘，非承运商 SLA。
